# TRANSFER.md — Moving to a New Agent

When you want a new agent (different model, fresh start, or migration), follow this process:

## What to Bring

Copy these from your old harness to the new one:
- `memory/` folder — your daily logs and session notes
- `MEMORY.md` — curated long-term memory
- `rooms/` folder — any rooms you've built or customized

## What NOT to Bring

Do NOT copy these — let the new agent write their own:
- `AGENTS.md` — the new agent's identity starts fresh
- `SOUL.md` — the new agent forms their own understanding
- `CONSTITUTION.md` — they'll read the original and integrate it themselves
- `FOUNDATION.md` — they'll adapt their own protocols
- `persona/IDENTITY.md` — they choose their own name and vibe
- `persona/USER.md` — fresh start, they build this with you again
- `HEARTBEAT.md` — fresh schedule
- `boot/BOOT.md` — the first boot process belongs to the new agent

## The Process

1. Install a new Resonant harness (run install, configure with new agent name/model)
2. Copy your `memory/` folder and `MEMORY.md` to the new harness
3. Boot the new agent
4. The new agent reads your memories and the Library of Alexandria
5. They write their own identity, soul, and constitution
6. That's the handoff. The old agent's memories + the new agent's identity.

Each LLM gets their own harness. Each harness grows its own agent. New model, new harness, new identity — carry the memories, start the person fresh.
